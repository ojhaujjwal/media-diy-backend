An Alchemy app on AWS is one **runtime** — usually a Lambda
Function — plus the **resources** it talks to: tables,
buckets, queues, streams. You wire them together with typed
**bindings**: call `S3.PutObject(bucket)` in your code and
Alchemy attaches the matching least-privilege IAM statement to
the function's role. The code is the policy.

New here? [Set up credentials](/aws/setup) first, then deploy
your first [Lambda](/aws/compute/lambda).

## Pick your runtime

- **[Lambda](/aws/compute/lambda)** — serverless, event-driven functions
  with public Function URLs. The primary documented path: every
  resource and event source in this section is shown
  running on Lambda.
- **[ECS](/aws/compute/ecs)** — long-running containers on Fargate.
  Alchemy bundles your Effect program into a Docker image and
  runs it as a Cluster + Service + Task.
- **[EC2](/aws/compute/ec2)** — full-control virtual machines, with the
  complete VPC networking toolkit around them.

Not sure which? See
[Choosing a runtime](/aws/compute/choosing-a-runtime).

## Data

- **[DynamoDB](/aws/data/dynamodb)** — key/value tables with
  `GetItem`/`PutItem` bindings and change-data-capture Streams.
- **[S3](/aws/data/s3)** — object storage with `GetObject`/`PutObject`
  bindings and bucket event notifications.
- **[RDS & Aurora](/aws/data/rds)** — managed Postgres/MySQL: the
  `Aurora` helper stands up the whole cluster in one call, with
  a `Connect` binding or the Data API at runtime.

## Messaging & events

- **[SQS](/aws/messaging/sqs)** — queues with a `SendMessage` binding and a
  `Stream`-shaped Lambda consumer.
- **[SNS](/aws/messaging/sns)** — pub/sub topics with a `Publish` binding
  and fan-out to queues and functions.
- **[EventBridge & Scheduler](/aws/messaging/eventbridge)** — event buses,
  rules, and cron/rate schedules that invoke your functions.
- **[Kinesis](/aws/messaging/kinesis)** — ordered, sharded data streams
  with a `PutRecord` binding and the same `Stream` consumer
  surface.

## Security & secrets

- **[Secrets & env](/aws/security/secrets-env)** — .env values via
  Config, Secrets Manager when the secret is shared, generated,
  or rotated.

## Observability

- **[CloudWatch](/aws/observability/cloudwatch)** — dashboards and metric
  alarms, declared in the same Stack as the resources they
  watch.

## Frontend & networking

- **[Websites](/aws/frontend/websites)** — the frontend block: static
  sites and built Vite apps on S3 + CloudFront as a single
  `StaticSite` resource.
- **[VPC & networking](/aws/networking)** — the `Network` helper
  and the VPC primitives, for when ECS or EC2 needs explicit
  networking.

## What are you building?

| You're building                       | Reach for                                                                     |
| ------------------------------------- | ----------------------------------------------------------------------------- |
| An HTTP API                           | [Lambda](/aws/compute/lambda) + Function URL + [DynamoDB](/aws/data/dynamodb)              |
| A typed HTTP API                      | [Effect HTTP API on Lambda](/aws/apis/effect-http-api)                      |
| A typed API for external clients      | [Effect RPC on Lambda](/aws/apis/effect-rpc)                                |
| Drive a MicroVM from a Lambda (internal RPC) | [MicroVMs](/aws/compute/microvms) + [Schemaless RPC](/apis/schemaless)   |
| A REST API with stages/custom domains | [API Gateway](/aws/apis/api-gateway)                                        |
| Your own domain on a site or API      | [Custom domains with Route53 + ACM](/aws/networking/custom-domains)               |
| A static site                         | [Deploy a static site](/aws/frontend/static-site) on S3 + CloudFront            |
| An event pipeline                     | [Kinesis](/aws/messaging/kinesis) → Lambda                                              |
| Background jobs                       | [SQS](/aws/messaging/sqs) + Lambda                                                      |
| Scheduled jobs                        | [EventBridge Scheduler](/aws/messaging/eventbridge) → Lambda                            |
| A Postgres database                   | [RDS & Aurora](/aws/data/rds)                                                      |
| Object processing                     | [S3 events](/aws/messaging/s3-events)                                            |
| Change data capture                   | [DynamoDB Streams](/aws/messaging/dynamodb-streams)                              |
| API keys or credentials for a function | [Secrets & env](/aws/security/secrets-env)                                            |
