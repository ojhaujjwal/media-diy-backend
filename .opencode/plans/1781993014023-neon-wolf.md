# Plan: Lint Rules & Agent Instructions to Prevent Common Mistakes

## Problem
During this session I repeatedly made several mistakes that existing rules should have caught, and several mistakes where no rule or guidance existed. The fixes for many issues required multiple iterations because I didn't follow established alchemy/Effect patterns.

## Mistakes Made & Root Causes

| Mistake | Count | Root Cause |
|---------|-------|------------|
| Used `as` type assertions | ~8 times | Didn't know alchemy's `Test.make` pattern exists; kept trying to bridge `Effect.runPromise` R=never gap manually |
| Used `Effect<any, ...>` | 2 times | Didn't know `Cloudflare.Worker` + `Cloudflare.Providers` types |
| Used non-existent `Layer.scoped` | 1 time | Didn't verify API before using |
| Wrong `Context.Service` syntax | 1 time | Didn't check reference |
| Created Layer-based test infrastructure that couldn't work | 1 time | Over-engineered instead of checking alchemy patterns first |

| Used `JSON.parse(JSON.stringify(...))` to bypass strict types | 1 time | Same — should have used Schema |
| Tried to exclude tests from lint | 1 time | Tried to relax rules instead of fixing root cause |

## Proposed Changes

### 1. AGENTS.md — Add Alchemy & Effect Guidance Sections

**A. Alchemy test patterns** (would have prevented 4+ mistakes):
```markdown
## Alchemy Test Patterns

- Use `Test.make` from `alchemy/Test/Vitest` for integration tests.
  Never manually call `Effect.runPromise` on alchemy stack effects.
- Pattern: `const { test, beforeAll, afterAll, deploy, destroy } = Test.make({ providers: Cloudflare.providers(), stage: "test" })`
- `const stack = beforeAll(deploy(Stack), { timeout: ... })`
- Access stack outputs via `yield* stack` in test effects.
- Reference: `alchemy-effect/packages/alchemy/src/Test/` for examples.

### Type Narrowing

- **Never use `as` type assertions.** Use `Schema.decodeUnknown` / `Schema.decodeUnknownSync`
  to parse `unknown` into concrete types.
- For test boundary values that intentionally violate types, test Schema validation
  directly (e.g., `Schema.decodeUnknown(MySchema)(invalidData)`) rather than trying
  to pass invalid data through typed APIs.
```

**B. Effect API verification** (would have prevented 2 mistakes):
```markdown
## Effect API Patterns

- Before using any Effect function, verify it exists by searching
  `repos/effect-smol/packages/effect/src/` for the function name.
  Never assume an API exists (e.g., `Layer.scoped` does not exist;
  use `Layer.effect` or `Layer.effectDiscard` instead).
- Services extend `Context.Service<Self, Type>()("identifier")` — not
  `Context.Service("identifier")<Self, Type>()`.
```

### 2. New Lint Rules (to add to `lint/rules/`)

**A. `no-async-await`** — Bans `async`/`await` and Promise constructors.
- Both AGENTS.md and effect-smol/AGENTS.md state: never use `async`/`await`.
- Catches: `async function`, `async () =>`, `await expr`, `new Promise(...)`
- Exception: `Effect.tryPromise` and `Effect.promise` are allowed (they adapt promises to Effect).

**B. `no-raw-date`** — Bans `new Date()` and `Date.now()`.
- Both AGENTS.md and effect-smol/AGENTS.md state: use `Clock` instead.
- Catches: `new Date()`, `new Date(...)`, `Date.now()`
- Test files are exempt (Date is fine for constructing test data).

- Register in `lint/index.ts`
