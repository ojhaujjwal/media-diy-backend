# Updated Plan: Integration tests for D1 & R2 via alchemy Test/Vitest + infra cleanup

## What changed from the original plan

The original plan (`1781952321857-quiet-star.md`) assumed alchemy@2.0.0-beta.51 + effect@4.0.0-beta.76 would work. Implementation revealed a **hard version incompatibility**: alchemy beta.51 calls `Schema.optional(Schema.Defect)` passing the *factory function* (no `.ast`), which crashes `SchemaAST.toType` at module load with `TypeError: Cannot read properties of undefined (reading 'encoding')`. This blocks every file that imports `alchemy/Cloudflare` (including `src/http/worker.ts`).

The implementation also accumulated a pile of fragile workarounds (pnpm patch + vitest alias + .mjs shim for `@effect/platform-node/NodeServices`; vitest alias + tsconfig paths for `alchemy/Test/Vitest`) that are all unnecessary once the versions are aligned.

**This plan replaces the original.** The primary change is a **version upgrade** to the latest mutually-compatible set, which unblocks the runtime crash and eliminates all workaround infrastructure. The remaining work (integration tests, infra cleanup, CI) follows the original plan's scope.
